# Stakeholder Map — MarkTask

> Ideation フェーズ / intent-capture ステージの成果物。
> 本プロジェクトは個人（パーソナルユース）が対象のため関係者は少数だが、役割を明示して意思決定の所在を明確にする。

## Key Stakeholders and Their Interests（主要ステークホルダーと関心事）

| ステークホルダー | 役割 | 主な関心事 |
|---|---|---|
| 本人（オーナー兼開発者兼ユーザー） | プロダクトオーナー / 開発者 / エンドユーザーを兼務 | 自分の日常タスク管理を置き換えられること。Markdown 自由記述、ロックイン回避、CLI 完結、Obsidian 連携、MCP 経由の AI 操作、繰り返し管理。 |
| 生成AI エージェント（間接的な利用主体） | MCP 経由の自動化アクター | MarkTask をツールとして安定操作できること（作成・参照・更新・完了の明確なインタフェース）。 |
| 将来の外部ユーザー / 開発者コミュニティ | 潜在的採用者（今回は対象外だが視野） | 導入障壁の低さ、オープンな形式、移行容易性。※MVP では最適化対象にしないが、設計判断がこの層を阻害しないことを意識。 |
| Obsidian（外部ツール・エコシステム） | 連携先（依存する外部制約） | vault 内で破綻なく扱える Markdown 形式・frontmatter・wikilink 互換性。MarkTask 側が Obsidian の規約に合わせる必要がある。 |

## Decision-makers vs. Influencers（意思決定者と影響者）

- **意思決定者（Decision-maker）**: 本人。スコープ・優先度・設計方針の最終決定権を持つ（本ワークフローの各承認ゲートの回答者）。
- **影響者（Influencers）**:
  - **Obsidian のファイル規約 / エコシステム** — 採用できるファイル形式・命名・リンク記法に制約を与える（合わせる側）。
  - **MCP 仕様** — 生成AI 連携インタフェースの形を規定する外部標準。
  - **個人のワークフロー原則**（open formats / Git・Markdown 優先 / API・CLI 第一級 / データはユーザーの資産）— 設計上の価値基準として全判断に影響。

## Communication Requirements（コミュニケーション要件）

- **意思決定の場**: 本 AI-DLC ワークフローの各ステージ承認ゲート。意思決定者は1名のため合意形成は速いが、**決定は artifact（本記録ディレクトリ配下）に明文化して単一の情報源とする**（後から自分が参照・再現できるように）。
- **外部制約の追跡**: Obsidian の Markdown/frontmatter/wikilink 仕様、MCP 仕様は「変更されうる外部依存」として feasibility / application-design で確認する。
- **将来公開に向けて**: 対象外ではあるが、判断の理由を記録に残し、後日コミュニティ公開時に説明可能な状態を保つ（個人プロジェクトでも意図を隠さない方針）。
