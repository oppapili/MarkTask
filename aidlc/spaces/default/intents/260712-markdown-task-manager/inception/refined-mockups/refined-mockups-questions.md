## Q4. 破壊的でない“移動”操作（delete=ソフト削除 / archive）の確認 UX は？

- A. 即実行（ソフト削除/退避なので確認なし。`list` から消えるが復旧可）＋実行結果に復旧方法を1行表示

- X. その他

## Q5. `list` の視認性（列と長文の扱い）は？

- 表示列（既定, 複数選択可）: A. status  B. due  C. priority  D. title  E. project  F. tags
- 長いタイトル: G. 端末幅で切り詰め（`…`）  H. 折り返さず全表示
- 件数が多い時: I. 全件表示（ページングなし, `less` 等に委ねる）  J. 既定で上限＋`--limit`
- X. その他
