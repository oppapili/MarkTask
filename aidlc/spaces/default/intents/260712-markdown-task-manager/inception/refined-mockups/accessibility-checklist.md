# Accessibility Checklist (CLI/MCP) — MarkTask

> Inception / refined-mockups 成果物。GUI が無いため WCAG を CLI/テキストUI 文脈に読み替えたチェックリスト。
> 上流: `../requirements-analysis/requirements.md`（requirements, NFR）, `design-system-mapping.md`。

## 1. 色・記号（色のみに依存しない）

- [x] ステータスは記号＋文字を必ず併記し、色は補助のみ（色覚特性に非依存 / Q1）。
- [x] `--no-color` を提供。非TTY（パイプ/リダイレクト）では色を自動オフ。
- [x] 情報を端末幅や罫線の見た目に依存させない（Unicode 罫線不使用、ASCII のみ）。

## 2. スクリーンリーダ / 支援技術

- [x] 出力はプレーンテキスト。装飾は行頭記号のみで、読み上げても意味が通る語（status 文字列）を併記。
- [x] エラーは記号だけでなく明確な文言で理由と次アクションを提示（例: 候補一覧、記法ガイド）。

## 3. 機械可読・自動化アクセシビリティ

- [x] `--json` で安定した機械可読出力（スクリプト/AI/支援ツールが解釈可能, Q2）。
- [x] Exit code 規約（0/1/2）で成否をプログラム判定可能。

## 4. 国際化・文字

- [x] UTF-8 前提。日本語タイトル/本文を破損なく表示・保存（NFR-4）。
- [x] 日付は ISO `YYYY-MM-DD` を基本表記（曖昧さ回避）。相対表記は補助（Q3）。

## 5. 予測可能性・一貫性（認知負荷）

- [x] コマンド間で記号・色・メッセージ・exit code を統一（design-system-mapping に単一定義）。
- [x] 破壊的に見える操作(delete/archive)は非破壊（ソフト）で、実行後に復旧手順を提示（誤操作からの回復可能性）。
- [x] 曖昧参照は勝手に決めず候補提示（overconfidence 回避 / FR-D5）。

## 6. 非対話・入力手段

- [x] すべての非破壊操作は非対話で完結（キーボード/パイプ/自動化フレンドリ）。
- [x] 確認プロンプトに依存しない（ソフト操作は即実行 / Q4=A）。

## Out of Scope（GUI 由来項目）

- キーボードフォーカス順序、ARIA、レスポンシブ・ブレークポイント、コントラスト比の厳密測定 — GUI が無いため非該当（`--no-color`＋記号併記でターミナルテーマ依存を回避）。
