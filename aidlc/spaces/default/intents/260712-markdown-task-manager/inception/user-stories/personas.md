# Personas — MarkTask

> Inception / user-stories 成果物。上流: `../requirements-analysis/requirements.md`, `../practices-discovery/team-practices.md`。
> 個人ツールのため実質2アクター。ストーリーは主に P1 視点、MCP 系は P2 視点で書く。

## P1｜開発者本人（CLI / Obsidian 利用者）— 主ペルソナ

- **役割**: MarkTask の唯一の人間利用者（＝作者）。日常タスクを管理する。
- **文脈**: Git / Markdown 中心のワークフロー。ターミナルと Obsidian を併用。Linux 環境。
- **ゴール**: タスクを素早くテキストで記述し、思考・メモを本文にそのまま残す。プレーンな Markdown 資産として長期保持し、ロックインを避ける。日常タスク管理を MarkTask に完全移行する。
- **ペインポイント**: 既存ツールが Markdown 自由記述に非対応／独自形式でロックインされる／繰り返しタスクの扱いが弱い。
- **重視**: CLI で完結すること、ファイルが Obsidian/エディタ/git でそのまま読めること、recurrence が破綻しないこと。

## P2｜生成AI エージェント（MCP クライアント）— 副ペルソナ（非人間）

- **役割**: MarkTask を MCP 経由で「ツール」として操作する生成AI。
- **文脈**: stdio ローカル・非公開の MCP サーバに接続。CLI と共通のコアを叩く。
- **ゴール**: タスクの作成・参照・更新・完了・検索・recurrence 設定を構造化された入出力で確実に行う。
- **ペインポイント**: 非対話環境なので曖昧な参照や破壊的操作の暗黙実行は困る（構造化・回復可能性が必要）。
- **重視**: 決定的な結果（file, status 等）、曖昧参照時の候補提示、ソフト削除による回復可能性。

## Priority Ranking

1. **P1（人間）** — 最優先。MVP の成功指標（完全移行・CLI 完結・ロックインゼロ）は P1 の体験で測る。
2. **P2（AI/MCP）** — 次点。MVP 成功指標「AI 操作可能」を満たすため主要操作を MCP で提供するが、機能はコア共有の薄いアダプタ。
